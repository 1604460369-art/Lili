# 设计规范 — 公文校对助手

## 1. 设计原则

- **简洁至上**：减少视觉噪音，让用户聚焦于校对任务
- **直观操作**：核心功能按钮醒目，流程自解释
- **专业可信**：配色和排版传递政务办公的专业感

---

## 2. 配色方案

以**淡蓝色**为主色调，白色为底色，灰度用作文本。

| 用途 | 颜色 | 色值 |
|------|------|------|
| 主色（导航栏/按钮） | 淡蓝 | `#4A90D9` |
| 主色悬停 | 深蓝 | `#357ABD` |
| 背景色 | 近白 | `#F5F8FC` |
| 卡片/输入区背景 | 纯白 | `#FFFFFF` |
| 边框 | 浅灰蓝 | `#D0DCE8` |
| 主文字 | 深灰 | `#333333` |
| 次要文字 | 中灰 | `#777777` |
| 错别字高亮 | 暖黄 | `#FFF3CD` |
| 语病高亮 | 浅橙 | `#FFE8D6` |
| 敏感词高亮 | 浅红 | `#FDE2E2` |
| 修改按钮 | 绿色 | `#5CB85C` |
| 成功提示 | 绿色 | `#27AE60` |
| 错误提示 | 红色 | `#E74C3C` |

## 4. 组件规范

### 4.1 按钮

| 类型 | 样式 |
|------|------|
| 主按钮（一键校对） | 淡蓝背景 `#4A90D9`，白色文字，圆角 6px，内边距 12px 24px |
| 修改按钮（采纳） | 绿色背景 `#5CB85C`，白色文字，小尺寸 |
| 危险按钮（不好用） | 浅色背景，灰色文字 |
| 悬停状态 | 所有按钮 hover 时颜色加深 10% |

### 4.2 文本输入区

- 宽度：占可用空间的 60%-65%
- 最小高度：400px
- 字体：微软雅黑 / 宋体，16px，行高 1.8
- 边框：1px solid `#D0DCE8`，圆角 8px
- 聚焦时边框变为 `#4A90D9`

### 4.3 错误高亮

- 错别字：背景 `#FFF3CD`，文字保持原色，底部红色波浪线
- 语病：背景 `#FFE8D6`，底部橙色波浪线
- 敏感词：背景 `#FDE2E2`，底部红色虚线下划线
- 悬停弹出提示框：白色背景，阴影，圆角，包含错误类型 + 建议

### 4.4 右侧问题列表

- 宽度：占可用空间的 35%-40%
- 每条问题：卡片式，白色背景，左边框颜色表示类型
- 点击可跳转到原文对应位置
- 每条右边有「采纳」按钮

### 4.5 反馈按钮

- 位于结果面板底部
- 👍 绿色 / 👎 灰色
- 点击后弹出简短感谢提示

## 5. 字体规范

| 层级 | 字体 | 大小 | 粗细 |
|------|------|------|------|
| 标题 | 微软雅黑 | 22px | Bold |
| 正文 | 微软雅黑 / 宋体 | 16px | Regular |
| 次要文字 | 微软雅黑 | 14px | Regular |
| 按钮文字 | 微软雅黑 | 15px | Medium |
| 高亮文字 | 继承正文 | 继承 | Regular |

---

## 6. 响应式说明

- 最小支持屏幕宽度：1024px
- 低于 1024px 时，右侧面板折叠到下方
- 移动端不做适配要求（主流使用场景在 PC）
