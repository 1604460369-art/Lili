# 技术方案 — 公文校对助手

## 1. 技术选型

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| 后端 | Python 3 + Flask | 轻量、易上手、Windows 兼容好 |
| 前端 | 单页 HTML + 内嵌 CSS/JS | 零依赖、打开即用、无需构建工具 |
| AI 服务 | DeepSeek API (Chat Completions) | 中文能力强、价格低、接口兼容 OpenAI 格式 |
| 敏感词匹配 | Python 本地字符串匹配 (AC 自动机) | 高效、离线、无需联网 |
| 配置管理 | JSON 文件 (config.json) | 简单直观，用户可直接编辑 |

## 3. API 设计

### 3.1 校对接口

```
POST /api/check
```

**请求：**
```json
{
  "text": "待校对的文本内容..."
}
```

**响应：**
```json
{
  "success": true,
  "issues": [
    {
      "id": 1,
      "type": "typo",           // 类型: typo=错别字, grammar=语病, sensitive=敏感词
      "position": { "start": 10, "end": 14 },
      "original": "原文片段",
      "suggestion": "修改建议",
      "replace_with": "建议替换为",
      "reason": "为什么建议修改"
    }
  ],
  "summary": {
    "total": 3,
    "typo": 1,
    "grammar": 1,
    "sensitive": 1
  }
}
```

**错误响应：**
```json
{
  "success": false,
  "error": "错误原因描述"
}
```

### 3.2 反馈接口

```
POST /api/feedback
```

**请求：**
```json
{
  "rating": "good",     // "good" 或 "bad"
  "text_length": 1234,
  "note": "用户备注（可选）"
}
```

**响应：**
```json
{
  "success": true,
  "message": "感谢反馈！"
}
```

## 4. DeepSeek API 调用方案

使用 DeepSeek Chat API (<https://api.deepseek.com/v1/chat/completions>)，接口兼容 OpenAI 格式。

**Prompt 设计思路：**
```
你是一位专业的公文校对专家。请检查以下文本中的问题：

1. 错别字：包括同音字、形近字误用（如"的/地/得"、"做/作"等）
2. 语病：搭配不当、句式杂糅、成分残缺
3. 用词规范：不符合政务公文写作规范的表述

请以JSON格式返回，每条问题包含：位置起止字符、原文、建议替换、理由。
注意：不要修改正确的部分，不要过度纠正。
```

## 5. 敏感词匹配方案

- 使用 `pyahocorasick` 库构建 AC 自动机
- 备选方案：如果安装 C 扩展失败，回退到 `re` 正则匹配
- 匹配策略：全词匹配 + 忽略标点符号
- 敏感词列表存储在 `sensitive-words.txt`，一行一词

## 6. 配置文件设计

`config.json`：
```json
{
  "api_key": "你的DeepSeek-API-Key",
  "api_url": "https://api.deepseek.com/v1/chat/completions",
  "model": "deepseek-chat",
  "max_text_length": 5000,
  "port": 5000,
  "debug": false
}
```
