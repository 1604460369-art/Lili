# 需求规格说明 — 公文校对助手

## 1. 产品概述

一款运行在 Windows 上的轻量级 AI 校对工具，面向政务/媒体用户，解决公文、新闻稿等正式文本中的错别字、语病、敏感词识别问题。

## 2. 目标用户

- **主要用户**：政务工作人员、媒体编辑
- **使用场景**：在发布/提交前，对公文段落、新闻稿件做最后校对
- **技术水平**：无需技术背景，会复制粘贴即可

## 3. 功能需求

### 3.1 文本输入
- **描述**：用户将待校对文本粘贴到输入区
- **输入区**：大文本区域（textarea），支持多段落文本
- **字数限制**：单次校对不超过 5000 字（API 成本控制）

### 3.2 一键校对
- **描述**：点击"一键校对"按钮，系统自动检查问题
- **检查内容**：
  - 错别字（如"的/地/得"误用）
  - 语病（搭配不当、句式杂糅）
  - 敏感词/不规范表述（基于用户提供的敏感词清单 + AI 判断）
- **处理流程**：前端发送文本 → 后端调用 DeepSeek API + 本地敏感词匹配 → 返回问题列表
- **响应时间**：通常在 3-8 秒内返回结果

### 3.3 错误高亮
- **描述**：在原文中高亮标注问题位置
- **高亮样式**：浅黄色背景 + 红色波浪下划线
- **交互**：鼠标悬停（hover）弹出提示框，显示错误类型和修改建议
- **右侧面板**：列出所有问题，点击可跳转到原文对应位置

### 3.4 修改建议
- **描述**：每个问题附带 AI 生成的修改建议
- **展示方式**：
  - 原文高亮区域：悬停显示简要建议
  - 右侧问题列表：显示完整建议和修改后文本预览

### 3.5 一键修正
- **描述**：用户确认后，点击按钮自动替换原文
- **粒度**：
  - 单项修正：每条建议旁有"采纳"按钮
  - 全部修正：一键采纳所有建议
- **可撤销**：修正后的改动支持撤销（Ctrl+Z 或撤销按钮）

### 3.6 用户反馈
- **描述**：用户可对校对结果进行简单评价
- **方式**：👍 好用 / 👎 不好用 两个按钮
- **存储**：反馈记录保存在本地文件中，包含时间戳和校对文本长度

## 4. 非功能需求

### 4.1 性能
- 校对响应时间 < 10 秒（5000 字以内）
- 界面操作流畅，无明显卡顿

### 4.2 可用性
- 无需安装复杂软件，双击启动脚本即可使用
- 界面简洁，核心流程不超过 3 步操作

### 4.3 可靠性
- API 调用失败时有友好提示，不崩溃
- 文本输入意外关闭时，内容应保留

### 4.4 安全性
- API Key 存储在本地配置文件，不上传至任何第三方
- 校对文本仅用于当次 API 调用，不在服务器端存储

### 4.5 兼容性
- 操作系统：Windows 10 / Windows 11
- 浏览器：Chrome / Edge（主流 Chromium 浏览器）

## 5. 验收标准

| 编号 | 测试场景 | 预期结果 |
|------|---------|---------|
| AC1 | 粘贴一段含错别字的公文，点击校对 | 正确识别错误位置并给出建议 |
| AC2 | 粘贴一段含敏感词的文本 | 敏感词被高亮标记 |
| AC3 | 鼠标悬停在错误高亮上 | 弹出修改建议 |
| AC4 | 点击"采纳"按钮 | 原文自动替换 |
| AC5 | 点击"全部修正" | 所有问题一次性修正 |
| AC6 | API 调用失败时 | 显示友好错误提示，不白屏 |
| AC7 | 点击 👍/👎 | 反馈记录写入本地文件 |
